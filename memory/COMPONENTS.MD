# COMPONENTS.md ‚Äì Predefined GitS UI Component Definitions

All components are named in the `S9-OV` (Section 9 Operative View) vocabulary.
Each entry defines: purpose, CSS classes, content structure, interactivity, and effects.

## 1. `PanelShell` ‚Äî Base Panel Container

**Status:** `implemented` ¬∑ Spec: `specs/panel-shell.md`

**Purpose:** Wraps every data panel. Provides border, header bar, holographic scanline body, and unified mount/alert/active state lifecycle for all child components.

**CSS Classes:**
```
.s9-panel                ‚Äî root container
.s9-panel__header        ‚Äî top label bar (height: --header-height)
.s9-panel__header-label  ‚Äî primary English label (Orbitron, uppercase)
.s9-panel__header-jp     ‚Äî Japanese counterpart label (required)
.s9-panel__header-status ‚Äî right-aligned status indicator (aria-live="polite")
.s9-panel__body          ‚Äî scrollable content area (.holographic for scanlines)
.s9-panel__footer        ‚Äî optional bottom status strip (height: --footer-height)

.s9-panel--active        ‚Äî brighter border + left cyan accent bar on __header
.s9-panel--alert         ‚Äî magenta border + alert-pulse animation (1Hz)
.s9-panel--booting       ‚Äî transient: crt-flicker animation; removed on animationend
.s9-panel--collapsed     ‚Äî body + footer hidden; header only (CSS-only, no animation)
```

**Required Content Structure:**
```html
<div class="s9-panel" role="region" aria-label="PANEL NAME"
     tabindex="0" data-s9-id="unique-id" data-s9-state="idle">
  <div class="s9-panel__header">
    <span class="s9-panel__header-label">PANEL NAME</span>
    <span class="s9-panel__header-jp">[Êó•Êú¨Ë™û]</span>
    <span class="s9-panel__header-status" aria-live="polite">‚óè ONLINE</span>
  </div>
  <div class="s9-panel__body holographic">
    <!-- child component content -->
  </div>
</div>
```

**JS API:** `components/panel-shell.js`
```js
import { mountPanel, unmountPanel, panelAlert, panelClearAlert, panelSetActive }
  from './components/panel-shell.js';
```
- `mountPanel(el)` ‚Äî add `--booting`, fire `s9:panel-mount` after `crt-flicker`
- `unmountPanel(el)` ‚Äî abort listener via AbortController
- `panelAlert(el, level, message)` ‚Äî add `--alert`, update status, fire `s9:panel-alert`
- `panelClearAlert(el)` ‚Äî remove `--alert`, restore status
- `panelSetActive(el, bool)` ‚Äî toggle `--active`, sync `data-s9-state`

**Custom Events (bubbles: true):**
- `s9:panel-mount` ‚Üí `{ id }`
- `s9:panel-alert` ‚Üí `{ id, level: 'warning'|'critical', message }`

**Effects:** Scanline overlay (`.holographic::after`), CRT flicker on mount (`crt-flicker` keyframe via `--booting`), hover/focus-within border glow, alert-pulse animation.

## 2. `DataStream` ‚Äî Live Scrolling Log

**Purpose:** Displays a continuous feed of timestamped events. Left column primary use.

**CSS Classes:**
```
.s9-stream              ‚Äî container (overflow-y: hidden)
.s9-stream__row         ‚Äî single log entry
.s9-stream__timestamp   ‚Äî `HH:MM:SS.mmm` left column, monospace, --text-dim
.s9-stream__source      ‚Äî source tag (e.g. "NET.DIV.6"), --neon-green
.s9-stream__message     ‚Äî event message, --text-primary
.s9-stream__row--alert  ‚Äî row with magenta highlight for warnings
```

**Content:** Entries scroll upward. New entries append at bottom, container auto-scrolls.

**Interactivity:** Click row to pin/expand. Hover to pause scroll. Filter by source via header dropdown.

**Effect:** New row fades in with glitch-enter animation. Alert rows pulse.

## 3. `NeuralReadout` ‚Äî Personnel / Entity Profile

**Purpose:** Displays biometric or entity data. Cybernetic identity card format.

**CSS Classes:**
```
.s9-neural              ‚Äî container
.s9-neural__id          ‚Äî large entity ID / designation
.s9-neural__ghost-line  ‚Äî "GHOST COEFFICIENT: 98.7%" readout
.s9-neural__avatar      ‚Äî image container (optional, clipped hexagonal)
.s9-neural__stats       ‚Äî data rows grid
.s9-neural__stat-label  ‚Äî left-aligned label
.s9-neural__stat-value  ‚Äî right-aligned monospace value
.s9-neural__status-bar  ‚Äî horizontal progress bar (neon fill)
```

**Required Content:** ID line + at least 4 stat rows + 1 status bar.

**Effect:** Avatar clipped with `clip-path: polygon(50% 0%, 100% 25%, 100% 75%, 50% 100%, 0% 75%, 0% 25%)`. Glitch on data update.

## 4. `ThreatMap` ‚Äî Global Visualization Panel

**Purpose:** Central area map/globe/spatial display. Primary hero visualization.

**CSS Classes:**
```
.s9-threatmap           ‚Äî container (position: relative, full center column)
.s9-threatmap__canvas   ‚Äî canvas or Three.js mount point
.s9-threatmap__overlay  ‚Äî CSS overlays atop the visualization (z-index: 10)
.s9-threatmap__node     ‚Äî interactive hotspot on the map
.s9-threatmap__node--active ‚Äî currently selected node (pulsing ring)
.s9-threatmap__crosshair ‚Äî targeting reticle overlay
.s9-threatmap__coords   ‚Äî lat/lon or coord readout strip
```

**Content:** Full-height canvas. Overlay includes corner brackets (ASCII-style), coordinate strip at bottom, node count in header.

**Interactivity:** Click nodes to select (triggers NeuralReadout update). Drag to pan. Scroll to zoom.

**Effect:** Bloom post-processing if Three.js. CSS fallback: radial glow on `.s9-threatmap__node--active`. Grid lines via `background-image: linear-gradient`.

## 5. `NetworkMatrix` ‚Äî Connection Graph Panel

**Purpose:** Displays network topology, communication routes, or dependency graph.

**CSS Classes:**
```
.s9-matrix              ‚Äî container
.s9-matrix__canvas      ‚Äî SVG or canvas element
.s9-matrix__node        ‚Äî graph node circle
.s9-matrix__node--root  ‚Äî primary node (larger, brighter)
.s9-matrix__edge        ‚Äî connection line (SVG line or canvas path)
.s9-matrix__edge--active ‚Äî highlighted active connection
.s9-matrix__label       ‚Äî node annotation
```

**Content:** SVG preferred for DOM interactivity. Canvas for high node counts (1000+).

**Effect:** Edges animate with a traveling dot along the path (SVG stroke-dashoffset animation). Node pulse on data activity.

## 6. `SystemTelemetry` ‚Äî Metric Gauges Panel

**Purpose:** System health: CPU, memory, network, threat level. Right column standard.

**CSS Classes:**
```
.s9-telemetry           ‚Äî container
.s9-telemetry__metric   ‚Äî individual metric row
.s9-telemetry__label    ‚Äî metric name + Japanese label
.s9-telemetry__bar      ‚Äî horizontal bar fill
.s9-telemetry__bar-fill ‚Äî colored fill element (width = % value)
.s9-telemetry__value    ‚Äî numeric readout right-aligned
.s9-telemetry__unit     ‚Äî unit label (--text-dim)
.s9-telemetry__threshold ‚Äî danger threshold marker line
```

**Color Rules:**
- 0‚Äì60%: `--neon-green`
- 61‚Äì85%: `--neon-amber`
- 86‚Äì100%: `--neon-magenta` + pulse

**Effect:** Bar fill animates on value change (`transition: width 0.4s steps(8)`). Threshold exceeded triggers panel-level alert state.

**CSS Classes:**
```
.s9-terminal            ‚Äî container

.s9-terminal__line      ‚Äî single output line

.s9-terminal__line--sys ‚Äî system response (--text-primary)

.s9-terminal__input-row ‚Äî input line container
.s9-terminal__prompt    ‚Äî prompt symbol `[S9]‚ñ∂`
.s9-terminal__input     ‚Äî text input element (transparent bg, neon cursor)
```

**Interactivity:** Command history (up/down arrows). Tab completion hook. Enter submits.

**Effect:** Typing triggers random 1-character glitch on nearby output lines (1% probability per keypress). Cursor blinks at 1.2s interval.

## 8. `AlertBanner` ‚Äî Critical Notification Strip

**Purpose:** Full-width or panel-width alert notification. Not a persistent component ‚Äî appears on event.

**CSS Classes:**
```
.s9-alert               ‚Äî container (position: fixed or absolute top)
.s9-alert--critical     ‚Äî magenta border + pulse
.s9-alert--warning      ‚Äî amber border
.s9-alert__icon         ‚Äî ‚öÝ or ‚¨° symbol
.s9-alert__code         ‚Äî alert code ID (e.g. "ERR-9C:0x4F")
.s9-alert__message      ‚Äî alert text
.s9-alert__timestamp    ‚Äî UTC timestamp
.s9-alert__dismiss      ‚Äî click to close
```

**Effect:** Enters with glitch animation (0.3s). Magenta border pulses at 1Hz. Auto-dismiss after 8s unless `data-persistent="true"`.

.s9-sequence__entry--pending   ‚Äî dim, dashed left border

.s9-sequence__step-time ‚Äî duration or timestamp
.s9-sequence__step-status ‚Äî status text

**Effect:** Active entry pulses. Step transitions animate with a 0.2s sweep. Failed steps flash twice before settling.

## 10. Layout: S9-OV (Section 9 Operative View)

**The standard three-column tactical configuration.**

```html
<div class="s9-ov">
  <header class="s9-ov__topbar"><!-- file/op name + tabs --></header>
  <div class="s9-ov__workspace">
    <aside class="s9-ov__left"><!-- DataStream, NeuralReadout --></aside>
    <main class="s9-ov__center"><!-- ThreatMap, NetworkMatrix --></main>
    <aside class="s9-ov__right"><!-- SystemTelemetry, SequenceLog --></aside>
  </div>
  <footer class="s9-ov__statusbar"><!-- global status, clock, user --></footer>
</div>
```

**Grid spec:**
```css
.s9-ov { display: grid; grid-template-rows: 36px 1fr 28px; height: 100vh; }
.s9-ov__workspace { display: grid; grid-template-columns: 280px 1fr 260px; gap: 4px; }
```
